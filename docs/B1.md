# B1: RelevanceTopK Router 实现文档

## 概述

B1 是一个非因果 relevance baseline router，用于与 M0 (ProductionSequentialRouter) 进行消融对比实验。

**核心特性：**
- 使用与 M0 完全相同的 candidate proposer
- 使用相同的 memory store snapshot 和 `top_k`
- 按 proposer 返回的 relevance ranking 顺序选择 memory
- **不调用** transfer critic
- **不计算** τ、η、LCB、UCB
- **不使用**风险门控
- 尊重 `max_shares_per_invocation` 预算限制

## B1 精确定义

### 接口

```python
@dataclass(frozen=True)
class RelevanceTopKRouterConfig:
    max_shares_per_invocation: int | None = None
    # 如果为 None，选择所有候选；如果指定，必须 >= 0

class RelevanceTopKRouter:
    router_name = "RelevanceTopKRouter"
    router_version = "1"
    
    def decide_from_proposal(
        self,
        *,
        receiver_agent_id: str,
        proposal: CandidateProposal,
        cards_by_id: dict[str, MemoryRoutingCard] | None = None,
        context: object | None = None,
        traversal_seed: int | None = None,
    ) -> RoutingResult: ...
    
    def decide(
        self,
        *,
        task: str,
        receiver_agent: str,
        candidates: list[CandidateTrace],
        cards_by_id: dict[str, MemoryRoutingCard],
        seed: int,
    ) -> tuple[list[RouterDecision], list[str]]: ...
```

### 决策逻辑

```python
# 选择 top-k 候选（按 proposer ranking）
if config.max_shares_per_invocation is None:
    share_limit = len(candidates)
else:
    share_limit = min(len(candidates), config.max_shares_per_invocation)

selected = proposal.ranked_candidates[:share_limit]
```

### 决策字段

| 字段 | 值 |
|------|-----|
| `action` | `"share"` 或 `"withhold"` |
| `reason` | `"relevance_topk_selected"` 或 `"relevance_topk_budget_exceeded"` |
| `decision_source` | `"relevance_topk_router"` |
| `candidate_position` | proposer ranking 中的位置 (0-indexed) |
| `score` | proposer 计算的 relevance score |
| `traversal_order` | 与 proposer ranking 一致（不随机打乱） |
| `tau_mean`, `tau_lcb`, etc. | `None`（B1 不使用 critic） |

## Router Factory

### 接口

```python
def build_router(
    mode: Literal["no-memory", "relevance-topk", "learned"],
    *,
    critic_checkpoint: str | Path | None = None,
    max_shares_per_invocation: int | None = None,
    seed: int = 0,  # 显式传入 traversal seed
    critic_config: SequentialRouterConfig | None = None,
) -> MemoryRouter:
```

### 模式映射

| Mode | Router | 需要 Checkpoint |
|------|--------|-----------------|
| `no-memory` | `NoMemoryRouter` (B0) | 否 |
| `relevance-topk` | `RelevanceTopKRouter` (B1) | 否 |
| `learned` | `ProductionSequentialRouter` (M0) | 是 |

### 约束

- `max_shares_per_invocation` 同时约束 B1 和 M0
- `learned` 模式缺少 checkpoint 时 **fail fast**（抛出 `ValueError`）
- 不允许 unknown mode 静默回退
- 不允许 `learned` 模式静默变成 `no-memory`
- 拒绝负数 share budget（`ValueError`）

## CLI 使用

### B0: NoMemoryRouter (默认)

```bash
python3 -m smtr.cli demo --seed 7 --top-k 4
```

### B1: RelevanceTopKRouter

```bash
python3 -m smtr.cli demo \
  --router-mode relevance-topk \
  --top-k 4 \
  --max-shares-per-invocation 3 \
  --seed 7
```

### M0: ProductionSequentialRouter

```bash
python3 -m smtr.cli demo \
  --router-mode learned \
  --critic-checkpoint checkpoints/critic_pi0.joblib \
  --top-k 4 \
  --max-shares-per-invocation 3 \
  --seed 7
```

### 新增 CLI 参数

| 参数 | 适用命令 | 说明 |
|------|----------|------|
| `--router-mode` | `demo`, `demo-real` | `no-memory` / `relevance-topk` / `learned` |
| `--critic-checkpoint` | `demo`, `demo-real` | critic 模型路径（`learned` 模式必需） |
| `--max-shares-per-invocation` | `demo`, `demo-real` | 每次路由最多共享的 memory 数量 |

```python
from smtr.router.baselines import RelevanceTopKRouter, RelevanceTopKRouterConfig
from smtr.runtime.graph import run_demo_with_repository
from smtr.memory.store import SQLiteSharedMemoryRepository
from smtr.memory.seed_memories import seed_repository

repository = SQLiteSharedMemoryRepository("memory.sqlite")
seed_repository(repository)

router = RelevanceTopKRouter(
    config=RelevanceTopKRouterConfig(max_shares_per_invocation=3)
)
state = run_demo_with_repository(
    repository=repository,
    seed=7,
    top_k=4,
    router=router,

```
